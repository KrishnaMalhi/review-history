# ReviewHistory — Monetization Strategy

---

## 💡 Core Principle

**Free until 10,000+ users.** Revenue kills growth if introduced too early.

A review platform's value is entirely dependent on the size and trust of its community. Monetising before reaching critical mass will:
- Drive away users who feel exploited
- Slow review growth (businesses on free tier feel less urgency to engage)
- Create conflicts of interest that damage platform credibility

**The Growth → Trust → Revenue sequence is non-negotiable.**

## 💳 Payment Integration Details

### JazzCash Integration

JazzCash is Pakistan's largest mobile wallet with 30M+ active users.

**Integration method:** JazzCash REST API (Sandbox → Production)

```
Business subscribes to Verified Badge
        ↓
Select JazzCash at checkout
        ↓
Enter JazzCash mobile number
        ↓
JazzCash sends MPIN prompt via USSD
        ↓
User confirms with JazzCash PIN
        ↓
Webhook POST to /payments/jazzcash/callback
        ↓
Verify payload hash (HMAC-SHA256)
        ↓
Activate subscription in database
        ↓
Send confirmation SMS to business
```

**Required credentials:**
- Merchant ID
- Password
- Integrity Salt (for HMAC verification)

### EasyPaisa Integration

EasyPaisa is Pakistan's second-largest mobile wallet with 25M+ users.

**Integration method:** EasyPaisa Payment Gateway API

```
Business selects EasyPaisa
        ↓
Redirected to EasyPaisa hosted payment page
        ↓
OTP sent to registered mobile number
        ↓
Payment confirmed
        ↓
EasyPaisa redirects to ReviewHistory callback URL
        ↓
Verify transaction via EasyPaisa API
        ↓
Activate subscription
```

### Stripe (International Cards)

For businesses paying with international credit/debit cards (diaspora Pakistani business owners):
- Stripe Checkout hosted page
- PKR supported as payment currency
- Webhooks for subscription lifecycle events

# 14 — Monetization Strategy

## 1. Five Core Revenue Streams

1. Claimed profile subscriptions
2. Sponsored listings
3. Local lead/performance ads
4. Premium consumer tools
5. Verification and data services

## 2. Example PKR Pricing Ladder

### Claimed Profiles
- Starter: PKR 1,500/month
- Growth: PKR 4,500/month
- Pro: PKR 12,000/month

### Premium Consumer
- Plus: PKR 299/month
- Pro Research: PKR 999/month

### Verification Services
- claim verification: one-time PKR 2,500+
- branch or document verification: custom

## 3. Year 1 Monetization Strategy

### Q1
No strong monetization. Build trust and content.

### Q2
Pilot profile claim subscriptions with local businesses and professionals.

### Q3
Add sponsored discovery placements with strict labeling.

### Q4
Test premium user tools and selective B2B reporting.

## 4. Revenue Sensitivity Example

Scenario:
- 200 starter accounts = PKR 300,000
- 50 growth accounts = PKR 225,000
- 10 pro accounts = PKR 120,000
- sponsored placements = PKR 150,000
- premium users = PKR 50,000

**Illustrative monthly total = PKR 845,000**

## 5. Anti-Corruption Rules

- never sell review deletion
- never sell trust-score changes
- clearly label ads
- keep organic ranking independent from paid products except labeled placements
